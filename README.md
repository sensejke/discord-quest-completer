# Quest Completer v2.5-claim-fix

Локальный userscript для запуска и управления Discord Quest-процессом через GUI в DevTools.

## Что делает

- Показывает встроенное окно управления квестами.
- Поддерживает основные типы задач:
  - `WATCH_VIDEO`
  - `PLAY_ON_DESKTOP`
  - `STREAM_ON_DESKTOP`
  - `PLAY_ACTIVITY`
  - `WATCH_VIDEO_ON_MOBILE` (если доступно в API)
- Запускает квесты по одному или в режиме авто-очереди.
- Отслеживает прогресс в реальном времени.
- Обрабатывает `429` (retry/pause), ведет счётчик и статус.
- Имеет базовый антиспам по логам и более безопасную обработку ошибок.

## Ограничения

- Desktop-квесты (`PLAY_ON_DESKTOP`, `STREAM_ON_DESKTOP`) в веб-версии Discord недоступны.
- Скрипт зависит от внутренних модулей Discord (`webpackChunkdiscord_app`) и может ломаться после обновлений клиента.
- Не гарантирует выполнение квестов, если Discord изменит API/валидацию.

## Быстрый старт

1. Открой Discord (Desktop или Web).
2. Открой DevTools → `Console`.
   - Windows/Linux: `Ctrl + Shift + I`
   - macOS: `Cmd + Option + I`
3. Вставь содержимое [`1.js`](./1.js) в консоль и нажми `Enter`.

## Интерфейс

- Вкладка `🎯 Квесты`
  - выбор квеста
  - `Старт` / `Стоп`
  - `🔄` авто-очередь
  - `↻` обновление списка
- Вкладка `📊 Стата`
  - выполнено/пропущено/время/429
  - статус детектора ошибок
- Вкладка `⚙️`
  - авто-пауза при частых `429`
  - порог `429` подряд
  - время паузы
  - умная сортировка

## Как работает антиспам

- Логи дедуплируются (повторы за короткое окно не выводятся).
- При частых `429` включается авто-пауза.
- Видео-прогресс отправляется с ограничением частоты, а не непрерывным флудом.

## Типичные проблемы

### `Ошибка: модули не найдены`

- Перезагрузи Discord (`Ctrl + R`) и запусти снова.
- Проверь, что запускаешь именно в клиенте Discord, а не в сторонней обертке.

### Квест есть, но не стартует

- Возможно, это desktop-only квест в вебе.
- Либо Discord поменял формат конкретной задачи.

### Много `429`

- Увеличь паузу в `⚙️` (`Ждать (сек)`).
- Отключи авто-очередь и запускай вручную.

## Безопасность и ответственность

- Скрипт использует внутренние интерфейсы Discord и не является официальным инструментом.
- Используй на свой риск.
- Не запускай код из непроверенных источников.
